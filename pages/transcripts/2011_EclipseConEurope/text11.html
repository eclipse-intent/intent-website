Notes:

We have one single document mixing both:

- explanations in natural language using Textile

- model fragments, defined with a generic textual syntax. Here I define an Epackage and a Eclass using the same syntax, but I can also define any element of any metamodel.

This structure allows to keep documentation close to the models it explains.

Consequently, we can easily determine :

- for a given model element, which part of the doc explains it

- for a given requirement, which models elements have been designed to fulfil it

- A very important feature of this syntax is the contribution concept, that allows to contribute to an element defined elsewhere.
