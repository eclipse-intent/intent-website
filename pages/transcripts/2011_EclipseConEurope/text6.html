Notes:

The most important thing to remember about Intent is that it allows to keep your doc synchronized with your technical artifacts.

When you make a change in the API of a Java plugin, the JDT automatically detects that a change occurred, and indicates with error markers the files in which this change lead to issues.

Well we want to do exactly the same with documentation : when you make a change in code that impacts the documentation, Intent will detects that a change occurred, and place synchronization errors in the documentation parts that may be impacted by this change.
