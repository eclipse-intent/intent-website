So we can see that if documentation is often discarded in Agile methods, it is because of its maintenance, which is both expensive and painful. That is exactly why the main purpose of Intent is to provide tooling for easily keeping your doc synchronized with your technical artifacts.

I don't know if you're familiar with the Ad-hoc Corrections pattern defined by Charles Weir, which tries to answer to the same problematic with a team best practice.
All the team shares a master copy of the documentation on paper. Any time a developer makes a change on the software that deviate from design (and hence requires a documentation update), he must write a correction in the margin of the master copy, indicating the documentation parts that have to be updated.

That's exactly what Intent synchronization mechanism is about : when you make a change in code that impacts the documentation, Intent will detect that a change occurred, find out all the documentations part related to this change and place synchronization errors on it.
