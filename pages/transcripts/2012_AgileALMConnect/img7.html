In order to achieve that goal, we defined the Literate Modeling approach.

It's based on Donald Knuth's Literate Programming theory. The main idea of this work is to mix explanations in natural language with code in a same document. With this structure, it is way easier to explain a part of code you have above your eyes, and to retrieve the doc related to a code part.

For the Literate Modeling approach, we have decided to use model fragments instead of code, as models are way more generic than code.
